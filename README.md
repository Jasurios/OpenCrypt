# OpenCrypt

> Безопасный SFTP-сервер с шифрованием файлов и автоматическим бэкапом

---

## Что такое OpenCrypt?

OpenCrypt — это самостоятельный SFTP-сервер, написанный на Go, предназначенный для безопасного хранения файлов с защитой на уровне криптографии. Каждый пользователь получает изолированное хранилище, защищённое уникальным мастер-ключом, который зашифрован паролем пользователя и никогда не хранится в открытом виде.

Проект подходит для:
- Организации приватного файлового хранилища на собственном сервере
- Безопасной передачи файлов между устройствами через SFTP
- Хранения чувствительных данных с автоматическим резервным копированием
- Создания многопользовательского файлового сервера с разграничением доступа

---

## Архитектура безопасности

Безопасность OpenCrypt построена на двухуровневой системе ключей:

**Пароль пользователя** хешируется через `argon2id` (64 МБ памяти, 4 потока) — один из самых надёжных современных алгоритмов хеширования паролей.

**Мастер-ключ** — случайный 32-байтный ключ AES-256, уникальный для каждого пользователя. Он шифруется производным ключом, выведенным из пароля через argon2, и хранится только в зашифрованном виде. Даже при компрометации базы данных `users.json` мастер-ключ без пароля расшифровать нельзя.

**Транспортный уровень** защищён SSH — соединение с сервером использует ключ `keys/server_key` (Ed25519).

**Изоляция пользователей** — каждый обычный пользователь видит только свою папку. Супер-пользователи имеют доступ ко всей директории файлов.

## Быстрый старт

### Требования

- Go 1.21 или новее
- OpenSSH (для `ssh-keygen`)

### Установка и первый запуск

```bash
# Клонировать репозиторий
git clone https://github.com/Jasurios/opencrypt.git
cd opencrypt

# Установить зависимости
go mod download

# Сгенерировать ключ сервера (см. раздел ниже)
ssh-keygen -t ed25519 -f keys/server_key -N ""

# Первый запуск — запустит мастер настройки
go run .
```

При первом запуске мастер настройки последовательно спросит:
- Путь до папки с файлами пользователей
- Порт сервера (например, `2022`)
- Папку для бэкапов
- Интервал бэкапа (`minute` / `day` / `month`)

После этого будет предложено создать первого пользователя.

## Создание SSH-ключа сервера

OpenCrypt использует Ed25519-ключ для идентификации сервера в SSH/SFTP-соединениях. Ключ должен находиться по пути `keys/server_key`.

**Создать новый ключ:**

```bash
ssh-keygen -t ed25519 -f keys/server_key -N ""
```

Флаг `-N ""` создаёт ключ без пароля (passphrase), что необходимо для автоматического запуска сервера.

В результате создаются два файла:
- `keys/server_key` — приватный ключ (никогда не передавайте третьим лицам)
- `keys/server_key.pub` — публичный ключ сервера

## Управление пользователями

Открыть меню управления:

```bash
go run . config
```

Доступные действия:
- Создать пользователя
- Изменить пароль (мастер-ключ автоматически перешифруется новым паролем)
- Изменить права (`super: yes/no`)
- Удалить пользователя

**Права пользователей:**

| Тип | Доступ |
|-----|--------|
| Обычный | Только своя папка (`USERSFILES/<имя>/`) |
| Супер | Вся директория `USERSFILES/` |

## Подключение к серверу

Подключиться к OpenCrypt можно любым SFTP-клиентом.

**Из командной строки:**

```bash
sftp -P 2022 username@your-server-ip
```

**FileZilla / WinSCP:**
- Протокол: SFTP
- Хост: адрес вашего сервера
- Порт: порт из `config.env` (по умолчанию `2022`)
- Имя пользователя и пароль: данные из `users.json`

---

## Конфигурация

Настройки хранятся в файле `config.env`:

```env
PORT=2022
USERSFILES=/home/user/files
BACKUPDIR=/home/user/backups
BACKUPINTERVAL=day
```

| Параметр | Описание |
|----------|----------|
| `PORT` | Порт SFTP-сервера |
| `USERSFILES` | Корневая директория файлов пользователей |
| `BACKUPDIR` | Папка для хранения бэкапов |
| `BACKUPINTERVAL` | Интервал бэкапа: `minute`, `day`, `month` |

Изменить настройки через меню:

```bash
go run . config
```

## Бэкапы

OpenCrypt автоматически создаёт ZIP-архивы с файлами пользователей. Архивы сжимаются алгоритмом Zstandard (`zstd`).

**Способы запуска бэкапа:**

Автоматически по расписанию — задаётся параметром `BACKUPINTERVAL` в конфиге.

Немедленно через меню:
```bash
go run . config
# Выбрать: [4] Сделать бэкап сейчас
```

Немедленно через триггер-файл — создайте файл `now.now` в папке пользователя. Watcher обнаружит его в течение минуты и запустит бэкап.

**Формат архива:** `backup_2006-01-02_15-04-05.zip`

После создания архива исходные файлы из папки пользователя удаляются, остаётся только архив и файл `backup_interval.txt`.
